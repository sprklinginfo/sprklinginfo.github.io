How will we get there?

OK it is time to think about how we get there? Frankly speaking, it will actually largely depend on what I will get from Library Systems/Services Assessment. Now I will try to outline several ideas that could potentially be on my real work plan to support one or more strategic Objectives.
First is to enhance web scale discovery. Are all our IR My Account or local collections included in Summon? Are there information silos? Evaluate our search results, like how relevant? Too many search results? what to show and what to suppress will be tough decision. Do we need include Open Access content like in Digital Commons? This will need work with other staff and faculty members. Can we use "Automated Contextual Research Assistance" through localized recommendations, research guides, and live reference help to improve UX? Do we need or is it possible to have social networking tools, like reviews, rating and sharing, features available in LibraryThing or BiblioCommons?

Enhance Web Scale Discovery

Supports: Objective 2.1, 1.3 & 2.3

  • Harvest/add records from IR or local collections
  • How relevant are search results?
  • Include Open Access content to Summon?
  • Automated Contextual Research Assistance
  • Extend to have social networking tools (reviews, rating, and reading list)?
This is nothing new. As information professionals, we all know the importance of metedata in our work. Here is a quote from a NISO paper... We have encountered numerous issues and challenges due to interoperability problem among disparate systems. Accurate and consistent metadata is the key to improve reliability and discoverability of our resources from now and into the future. Closely monitor Linked data in libraries will be another important task relates to metedata and cataloging.

Metadata Quality Assessment & Control

Mobilizing the library

I consider this is a challenged area in a healthy way. Based on my web development work experience, this can be as simple as to make the website responsive or adaptive to fit different device screens. But it can also be a complicated and huge project. Mobile friendly solution is a must. It means our users can access library resources / services from any devices. So what is our mobile strategy? Should we go mobile web or native web app? Each has its own pros and cons. I did a simple test on Sheridan website to access the library page on my android phone. Here are some results: 1)Website is responsive but the library link is pushed down to the bottom, hard to locate 2)the ‘Search’ icon on top right corner is not for searching library resources

Supports: Objective 2.1 & 2.3

  • Access library resources & services from any devices
    • Better user experience, increase circulation and student engagement
  • What is our mobile strategy?
    • Mobile/Responsive Website vs. Mobile App
    • Optimized User Experience/Content/Page Speed/SEO
  • Intensive and ongoing collaboration with campus IT

Mobilizing the library

Supports: Objective 2.1 & 2.3

Embed library resources and services in LMS

Usually library resources like articles, books, and web resources are added to courses. Can we create a module or course for library instructional resources or services? Optional and self-paced, Like MOOC? How to seamlessly integrate LMS and E-Reserves system. Integrate with eResearves system for more resources like scanning books or articles, copyright assistance and alert etc.

Supports: Objective 2.1 & 2.2

  • Need more research on OPAC, Summon APIs and LMS in use
  • A module/course for library instructional resources and services?
  • Integrate Course Reserves/eReserves system for more resources
  • Challenge on Authentication
  • Collaboration with librarians, copyright staff, and campus IT
