Action 1 -- Library Systems/Services Assessment

Naturally I consider my first work action is “Library Systems/Services Assessment”. It supports all objectives. I see this as a long-time, multi-dimensional, ongoing and iterative process. Starts by gathering data or reports produced by library team, IT or other units on campus for library website, ILS, Discovery layer, Institutional Repository and other technologies currently being used. Most likely we need conduct some new assessments if no data or reports available in some areas. The assessments that I am interested include User Research to understand user behaviors, needs and motivations and Service quality assessment. (Data collection and analysis methods/User Research: Card Sorting, interviews, focus group, persona, survey, usability testing, task analysis and user cases, google analytics etc., Service quality assessment like LibQUAL etc.)

Supports: All objectives

  • A long-time, multi-dimensional, ongoing and iterative process
  • Gathering data (Reports and data by Library team, IT & other units on campus)
  • Library systems (Website, ILS, Discovery, IR & other technologies)
  • User Research to understand user behaviors, needs and motivations
  • Service quality assessment

How will we get there?

OK it is time to think about how we get there? Frankly speaking, it also depends on what I will get from Library Systems/Services Assessment. Now I will try to outline several ideas that could potentially be on my real work plan to support one or more Work Objectives.
Start by enhancing web scale discovery. It supports Objective 2.1, 1.3 and 2.3. It is a journey to find answers to a series of questions. Are all our IR records or local collections included in Summon? If not, add to catalog or harvest them. Are search results relevant? Too many search results? what to show and what to suppress will be a tough decision. Do we need pick and include related Open Access content to our discovery? like content in Digital Commons, so users can find those content without leaving our site. Summon has Automated Contextual Research Assistance feature, so we can include localized recommendations, research guides, and live reference help to improve UX. Finally do we need social networking tools, things like reviews, rating and sharing reading list? Those are common features available in LibraryThing or BiblioCommons.

Action 2 -- Enhance Web Scale Discovery

Supports: Objective 2.1, 1.3 & 2.3

  • Harvest/add records from IR or local collections
  • Are search results relevant
  • Include Open Access content to Summon?
  • Localized recommendations, research guides, and live reference help to improve UX
  • Extend to have social networking tools (reviews, rating, and sharing reading list)
Now let's think about metadata. It supports Objective 2.1 & 1.3. This is nothing new. As information professionals, we all know the importance of metadata in our work. Here is a quote from NISO ... We have encountered numerous issues and challenges due to interoperability problem among disparate systems. Accurate and consistent metadata is the key to improve reliability and discoverability of our resources from now and to the future. Semantic Web and Linked Data is the future of our information society. Closely monitor Linked data in libraries will be another important task relates to metadata and cataloging area.

Action 3 -- Metadata Quality Assessment & Control

"Metadata is key to ensuring that resources will survive and continue to be accessible into the future."
(NISO, 2012, p. 1)

Supports: Objective 2.1 & 1.3

  • Interoperability issue prevents us from discovering and sharing resources
  • Accuracy and consistency is the key to improve reliability and discoverability of our resources
  • Closely monitor Linked data / Semantic Web movement (RDA/BIBFRAME)

Action 4 -- Mobilizing the library

Now let's talk about mobilizing the library. It supports Objective 2.1 & 2.3. I consider this is a challenged area but in a healthy way. Based on my web development experience, this can be as simple as to make the website responsive or adaptive to fit different device screens. But it can also be a very complicated and huge project. Nowadays mobile friendly solution is a must. It means our users can access library resources / services from any devices. It has numerous benefits to library...just to name a few. But the most important thing is: what is our mobile strategy? Should we go mobile web or native mobile app? Each has its own pros and cons. No matter what we will decide, a truly mobile friendly solution should have User Experience and content optimized to mobile devices. This work also requires intensive and ongoing collaboration with campus IT. Curiously I did a simple test on Sheridan library page on my android phone last weekend. Here are some results I’d like to share …

Supports: Objective 2.1 & 2.3

  • Access library resources & services from any devices
    • Better user experience, increased circulation and student engagement
  • What is our mobile strategy?
    • Mobile/Responsive Website vs. Mobile App
    • Optimized User Experience/Content/Page Speed/SEO
  • Intensive and ongoing collaboration with campus IT

Action 4 -- Mobilizing the library

Supports: Objective 2.1 & 2.3

  • The ‘Search’ icon on top right corner is not for searching library resources(confusing)
  • Scroll down to find the ‘Summon’ link, touch to see the search box, then type keywords to search (too many steps required)

Action 5 -- Embed library resources and services in LMS

Last but not least, let’s talk about LMS. It supports Objective 2.1 & 2.2. Usually library resources like articles, books, and web resources are added or embedded to course pages in LMS. We also can create a module...to be included into other courses. or even create a optional, self-paced online course, you know, a little bit like MOOC. I want to do more research on OPAC, Summon APIs and LMS currently used in the library, to identify any gaps or opportunities to integrate them together, and to integrate with eResearves system for more resources and services like scanning book chapters or articles, copyright assistance and alert etc. in LMS. Since we need work on 3 or more systems, the authentication will be a big challenge. It also requires collaborative work among the library team, CTL and campus IT.

Supports: Objective 2.1 & 2.2

  • A module for library instructional resources and services, or a MOOC like course
  • More research on OPAC, Summon APIs and LMS
  • Integrate Course Reserves/eReserves system for more resources
  • Challenge on Authentication
  • Collaborative work among whole library team, CTL and campus IT

Projects Showcase

OK, Enough dry talk. Let's have some fun together. I want to give you a short tour on a couple of projects I did or am doing now. I hope you will enjoy it.

Islandora Oral Histories Solution Pack

To preserve audio, video materials, sync-play transcript and translation, and annotate those materials for collaborative research on Islandora/Fedora stack

Main technologies: Drupal custom module development, Islandora, Fedora, HTML5, JavaScript, ReactJS, PHP, RDF/XML, Apache Solr, XSLT, REST APIs, MODS, DC, emerging W3C Web Annotation Data Model and WEBVTT spec

Islandora Oral Histories Solution Pack

Islandora Living Research Lab Solution Pack

Custom developed to power Learning in Neural Circuits (LINC) project, a new repository and virtual research environment for the Department of Biology to institute best practices for research data management by replacing traditional paper and spreadsheet based system.

Main technologies: Drupal custom module development, Islandora, Fedora, HTML5, JavaScript, PHP, XML, Apache Solr, XSLT, REST APIs, MODS, DC, Darwin Core

Islandora Living Research Lab Solution Pack

CNIB Library Integration Project (CLIP)

Infor web portal and discovery solution Iguana. Online catalogue search, manage loans and holds, and create readinglist, online reads, online DAISY book download etc. In the article Posted On October 6, 2015
  • Migrated and implemented new V-smart ILS
  • Built CNIB Library website on Iguana Discovery System
  • Fully accessible (WCAG 2.0 AA and AAA standards) and bilingual
  • Custom developed features on Iguana: Browse by Category, and High Contrast Switch
  • An example site used on a blog post "Four Discovery Services to Watch"(infotoday.com)

DSU Technology Stack

Installed, configured and maintained Fedora Commons Repository application stack, Islandora and Drupal multi-site platform on multiple Linux severs to support all DSU digital collections and projects. This technology stack now hosts about 19 Drupal sub websites/projects with master website at http://digitalscholarship.utsc.utoronto.ca.

Redmine Project Management System

DSU desperately needed a project management system to facilitate the fast growing projects. We decided to install Redmine after research and evaluation. Redmine is a free and open source, web-based project management and issue tracking tool, built on Ruby on Rails, Highly customizable, flexible user management, issues/ticket system, time tracking, wiki, documents, news/blog, forums etc., extensible with plugins (email to create ticket, agile board, project tree view, document management, etc)
  • Free and open source, web-based project management and issue tracking tool
  • Must-have tool for DSU day-to-day operation
  • Home for over 30 projects
